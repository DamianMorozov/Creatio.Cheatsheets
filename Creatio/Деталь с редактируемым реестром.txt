1. Создать схему объекта детали.
  [Конфигурация] - [Схемы] - [Добавить] — [Объект]
  UsrCourierService / Курьерская служба / Базовый объект ( Base )
  + справочную колонку UsrOrder / Заказ. Обязательная для заполнения: На уровне приложения.
  + справочную колонку UsrAccount / Контрагент. Обязательная для заполнения: На уровне приложения.
------------------------------------------------------------------------------------------------------------------------
2. Создать и сконфигурировать схему детали.
  [Конфигурация] - [Схемы] - [Добавить] — [Схема модели представления карточки]
  UsrCourierServiceDetail / Схема детали "Курьерская служба в заказе" / Базовая схема детали с реестром ( NUI ) | 4 в списке
  // Базовая схема детали с реестром (BaseGridDetailV2)
  LocalizationStrings: Caption
  Для того чтобы сделать реестр детали редактируемым, в ее схеме необходимо:
  - Добавить зависимости от модулей ConfigurationGrid, ConfigurationGridGenerator, ConfigurationGridUtilities.
  - Подключить миксин ConfigurationGridUtilities.
  - Установить для атрибута IsEditable значение true.
  - В массив модификаций diff добавить конфигурационный объект, в котором устанавливаются свойства и выполняется привязка методов-обработчиков событий реестра детали.
------------------------------------------------------------------------------------------------------------------------
3. Выполнить настройку детали в замещающей схеме страницы редактирования раздела.
  Создайте замещающий клиентский модуль, в котором в качестве родительского объекта укажите [Страница редактирования заказа] ([Order edit page], OrderPageV2).
------------------------------------------------------------------------------------------------------------------------
4. Выполнить настройку колонок реестра детали.
  В общем случае регистрация детали с редактируемым реестром в системе не нужна. Однако, если по каким-либо причинам деталь все же нужно зарегистрировать в системе, 
  то для этого выполните следующий SQL-сценарий.
